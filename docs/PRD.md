## 1. 产品概述

### 1.1 产品定位

WMS 出库单转换工具是一款面向仓储物流场景的 **PDF/Excel 格式转换 Web 应用**，用于将各商家不同格式的出库单（PDF 或 Excel）快速转换为标准 OMS（订单管理系统）出库 Excel 格式，实现多商家模板一键转换，提升仓储出库单处理效率。

### 1.2 核心价值

| 价值维度 | 说明 |
|---------|------|
| **效率提升** | 替代人工手动复制粘贴，自动化解析 PDF/Excel 出库单，一键生成标准 OMS 格式 |
| **多模板适配** | 内置 3 种商家模板（黔寨寨、黎明屯、欢乐牧场），支持按模板自动路由解析逻辑 |
| **智能拆零** | 根据商品拆零配置自动分配数量到二级单位或最小单位，减少人工判断 |
| **数据可追溯** | 每次转换自动记录日志，支持查询统计和结果文件下载 |
| **多人协作** | Web 化部署，支持多人同时使用，API 限流防止滥用 |

### 1.3 技术架构

```
前端 (React + Vite)  ←→  FastAPI 后端  ←→  SQLite 数据库
        ↓                      ↓                  ↓
   拖拽上传界面           PDF/Excel解析       拆零配置存储
   拆零管理页面           转换核心逻辑        订单号序列管理
   日志查看页面           转换日志记录        欢乐牧场历史订单
```

### 1.4 输出格式标准

所有模板统一输出为标准 `OMS出库.xlsx` 格式，共 10 列：

| 列 | 字段名 | 说明 |
|----|--------|------|
| A | 单据编号 | 从源文件提取或系统自动生成 |
| B | 商户编码 | 模板默认值或用户自定义 |
| C | 供货机构 | 固定值 `ZTOWHHY001` |
| D | (空) | 预留列 |
| E | 收货人信息 | 格式：`收货人,电话,地址` |
| F | 收货机构/店铺名 | 从源文件提取 |
| G | 商品名称 | 从源文件提取 |
| H | 商家商品编码 | 从源文件提取 |
| I | 二级单位数量 | 拆零时填入 |
| J | 最小单位数量 | 不拆零时填入 |

## 2. 用户角色说明

### 2.1 角色定义

| 角色 | 描述 | 主要使用场景 |
|------|------|-------------|
| **仓储操作员** | 日常处理出库单转换的一线工作人员 | 上传文件、选择模板、执行转换、下载结果 |
| **拆零管理员** | 负责维护商品拆零规则的管理人员 | 新增/编辑/删除拆零配置、批量保存 |
| **系统管理员** | 负责系统部署和监控的运维人员 | 查看转换日志、排查问题、配置模板 |

### 2.2 用户画像

**仓储操作员 - 小李**
- 每天需要处理 20-50 份出库单
- 痛点：手动转换耗时易错，不同商家格式不统一
- 期望：上传文件后一键转换，自动下载结果

**拆零管理员 - 王主管**
- 负责维护商品拆零规则库
- 痛点：规则变更频繁，需要快速更新
- 期望：页面内直接编辑，批量保存，即时生效

## 3. 功能需求列表

### 3.1 文件转换功能（核心）

#### 3.1.1 功能描述

用户上传 PDF 或 Excel 格式的出库单，系统根据所选模板自动解析并转换为标准 OMS 出库 Excel 格式。

#### 3.1.2 业务流程

```
1. 用户选择模板（黔寨寨/黎明屯/欢乐牧场）
2. 用户上传文件（支持多文件，最多50个）
3. 前端校验文件类型和大小
4. 后端流式接收文件
5. 根据模板调用对应解析器
6. 解析头部信息和商品明细
7. 应用拆零规则路由数量
8. 生成标准 OMS Excel 文件
9. 返回转换结果和下载链接
10. 前端自动触发下载
11. 记录转换日志
```

#### 3.1.3 逻辑细节

| 环节 | 规则 |
|------|------|
| **文件校验** | 单文件最大 50MB，单次最多 50 个文件，文件扩展名必须匹配模板要求 |
| **模板路由** | `qzz` → PDF 解析器，`lmt` → 黎明屯 Excel 解析器，`hlmc` → 欢乐牧场 Excel 解析器 |
| **商户编码** | 优先使用用户输入，为空时使用模板默认值，最大 64 字符，仅允许字母/数字/下划线/连字符 |
| **拆零校验** | 仅黎明屯模板强制校验：商品编码必须存在于拆零配置表，缺失时阻断并提示 |
| **数量路由** | 黔寨寨：全部放最小单位列；其他模板：根据拆零配置决定放二级单位或最小单位列 |
| **文件命名** | 格式：`{模板名称}_{YYYYMMDD}.xlsx`，重名时自动追加序号 |
| **结果清理** | 下载文件保留 24 小时后自动清理 |

#### 3.1.4 模板详情

| 模板 | 输入格式 | 商户编码 | 解析策略 | 拆零校验 |
|------|----------|----------|---------|---------|
| 黔寨寨贵州烙锅 | PDF | Q20260427013 | 提取 PDF 文本和表格，正则匹配头部字段 | 不校验 |
| 黎明屯铁锅炖 | Excel | Q20260427017 | 搜索关键词定位表头，提取收货机构 | **强制校验** |
| 欢乐牧场 | Excel | Q20260427015 | 支持两种格式：店铺列式/门店名称行式，自动按店铺拆分 | 不校验 |

#### 3.1.5 欢乐牧场特殊逻辑

1. **店铺匹配**：根据店铺名称关键词匹配预设收件人信息（名称、电话、地址）
2. **订单号生成**：格式 `{店铺缩写}{日期}{4位序号}`，如 `JQ260506001`
3. **店铺缩写映射**：金桥→JQ、银泰→YT、金银潭/金银屯→JYT
4. **防重复机制**：基于商品列表签名判断是否同一订单，避免重复生成订单号
5. **默认收件人配置**：可通过环境变量 `HLMC_RECEIVERS_JSON` 或 `config.json` 自定义

#### 3.1.6 API 限流

| 参数 | 值 |
|------|-----|
| 时间窗口 | 60 秒 |
| 最大请求数 | 30 次 |
| 超限响应 | HTTP 429，提示"请求过于频繁，请稍后再试" |

### 3.2 商品拆零管理功能

#### 3.2.1 功能描述

提供页面化管理商品拆零规则，支持增删改查和批量保存，数据持久化到 SQLite 数据库。

#### 3.2.2 业务流程

```
1. 用户进入拆零管理页面
2. 系统从数据库加载所有拆零配置
3. 按创建时间倒序展示
4. 用户可搜索、筛选、分页浏览
5. 用户可新增记录（默认拆零=是）
6. 用户可编辑商品编码和拆零规则
7. 用户可删除单条记录
8. 点击保存批量提交变更
9. 系统校验后写入数据库
10. 刷新列表显示最新数据
```

#### 3.2.3 逻辑细节

| 功能 | 规则 |
|------|------|
| **数据加载** | 启动时从 `商品拆零模板.xlsx` 种子导入（仅新增，不覆盖已有） |
| **新增记录** | 自动生成 ID，创建时间由数据库自动生成，默认拆零=是 |
| **编辑规则** | 修改商品编码或拆零值后标记为待保存状态，显示未保存提示 |
| **批量保存** | 仅提交变更记录，支持混合新增和更新操作 |
| **删除确认** | 弹窗确认，区分未保存记录（仅本地移除）和已保存记录（数据库删除） |
| **返回保护** | 有未保存更改时返回需二次确认 |
| **搜索筛选** | 支持按商品编码模糊搜索，按拆零规则筛选（全部/拆零/不拆零） |
| **分页控制** | 默认每页 10 条，支持 10/20/50 条切换 |

#### 3.2.4 数据表结构

```sql
CREATE TABLE split_codes (
    code TEXT PRIMARY KEY COLLATE NOCASE,  -- 商品编码（不区分大小写）
    split TEXT NOT NULL DEFAULT '是',       -- 拆零规则：是/否
    item_name TEXT,                         -- 商品名称（预留）
    created_at TEXT NOT NULL DEFAULT (datetime('now', 'localtime'))  -- 创建时间
);
```

### 3.3 转换日志功能

#### 3.3.1 功能描述

自动记录每次转换操作的详细信息，支持查询、筛选和结果文件重新下载。

#### 3.3.2 日志字段

| 字段 | 类型 | 说明 |
|------|------|------|
| timestamp | string | 转换时间（ISO 格式） |
| template_key | string | 模板标识（qzz/lmt/hlmc） |
| template_name | string | 模板名称 |
| file_names | array | 上传文件名列表 |
| file_count | number | 上传文件数量 |
| item_count | number | 解析商品总数 |
| store_count | number | 涉及店铺数量 |
| total_quantity | number | 商品合计数量 |
| stores_list | array | 店铺名称列表 |
| output_filename | string | 输出文件名 |
| status | string | 转换状态（success/error） |
| error | string | 错误信息（失败时） |

#### 3.3.3 逻辑细节

| 功能 | 规则 |
|------|------|
| **存储格式** | JSONL 文件（每行一条 JSON 记录） |
| **查询限制** | 默认返回最近 200 条 |
| **状态筛选** | 支持全部/成功/失败三种状态 |
| **搜索功能** | 支持按模板名称或文件名模糊搜索 |
| **自动刷新** | 可开启 30 秒自动刷新 |
| **错误展开** | 失败记录可展开查看错误详情 |
| **结果下载** | 成功记录可直接重新下载输出文件 |
| **分页展示** | 每页 20 条 |

### 3.4 系统管理功能

#### 3.4.1 配置管理

| 配置项 | 说明 | 配置方式 |
|--------|------|---------|
| 模板定义 | 模板名称、接受格式、默认商户编码 | `config.json` 或代码默认值 |
| 欢乐牧场收件人 | 店铺匹配规则（名称、电话、地址） | `config.json` 或环境变量 |
| CORS 来源 | 允许的前端域名 | 环境变量 `WMS_CORS_ORIGINS` |
| 文件限制 | 最大文件大小、最大文件数量 | 代码常量 |
| 下载 TTL | 下载文件保留时间 | 代码常量（默认 24 小时） |
| 限流策略 | 时间窗口、最大请求数 | 代码常量 |

#### 3.4.2 安全机制

| 机制 | 实现方式 |
|------|---------|
| 路径遍历防护 | 下载文件名校验，拒绝包含 `/`、`\`、`..` 的文件名 |
| API 限流 | 基于 IP 的滑动窗口限流 |
| CORS 限制 | 严格限定允许的来源域名 |
| 商户编码校验 | 正则校验，仅允许字母/数字/下划线/连字符 |
| 流式上传 | 分块读取，避免大文件内存溢出 |

## 4. 数据实体关系

### 4.1 实体定义

```
┌─────────────────┐
│   Template      │  模板
├─────────────────┤
│ template_key    │  模板标识（PK）
│ template_name   │  模板名称
│ accept          │  接受文件格式
│ merchant_code   │  默认商户编码
└─────────────────┘

┌─────────────────┐
│   SplitCode     │  商品拆零配置
├─────────────────┤
│ code            │  商品编码（PK）
│ split           │  拆零规则（是/否）
│ item_name       │  商品名称
│ created_at      │  创建时间
└─────────────────┘

┌─────────────────┐
│   HLMCSequence  │  欢乐牧场订单序号
├─────────────────┤
│ date            │  日期（PK）
│ prefix          │  店铺缩写（PK）
│ seq             │  当前序号
└─────────────────┘

┌─────────────────┐
│   HLMCHistory   │  欢乐牧场订单历史
├─────────────────┤
│ id              │  自增ID（PK）
│ shop_name       │  店铺名称
│ date            │  日期
│ signature       │  订单签名（UNIQUE）
│ order_no        │  订单编号
└─────────────────┘

┌─────────────────┐
│   ConversionLog │  转换日志
├─────────────────┤
│ timestamp       │  转换时间
│ template_key    │  模板标识
│ file_names      │  文件名列表
│ item_count      │  商品数量
│ status          │  转换状态
│ error           │  错误信息
│ ...             │  其他统计字段
└─────────────────┘
```

### 4.2 实体关系图

```
Template (1) ──────< (N) ConversionLog
    │                      │
    │ 模板定义              │ 记录每次转换
    │                      │
    ▼                      ▼

SplitCode (1) ──────> (N) ConversionLog.item
    │
    │ 拆零规则
    │ 路由数量
    ▼

┌──────────────────────────────────────────────────┐
│              转换流程数据流                        │
│                                                  │
│  UploadFile ──→ Parser ──→ SplitRule ──→ Output  │
│     │              │            │           │     │
│     │              │            │           │     │
│  校验格式      解析头部       查询拆零     生成     │
│  限制大小      提取商品       路由数量     Excel   │
│  UUID命名      匹配店铺       分配列位     下载    │
└──────────────────────────────────────────────────┘

HLMCSequence (1) ──────< (N) HLMCHistory
    │                           │
    │ 序号生成                   │ 订单记录
    │ 防重复                     │ 签名校验
    ▼                           ▼

┌──────────────────────────────────────────────────┐
│              欢乐牧场订单号生成逻辑                 │
│                                                  │
│  1. 计算商品列表签名                              │
│  2. 查询 HLMCHistory 是否有相同签名                │
│  3. 若存在 → 返回已有订单号                        │
│  4. 若不存在 → 从 HLMCSequence 获取新序号          │
│  5. 生成订单号 → 写入 HLMCHistory                  │
└──────────────────────────────────────────────────┘
```

### 4.3 关系说明

| 关系 | 说明 |
|------|------|
| **Template → ConversionLog** | 一对多：一个模板可对应多次转换记录 |
| **SplitCode → ConversionLog.item** | 一对多：一个拆零规则可影响多个商品行的数量分配 |
| **HLMCSequence → HLMCHistory** | 一对多：一个日期+店铺组合对应多个历史订单 |
| **UploadFile → Parser** | 一对一：每个文件根据模板类型路由到对应解析器 |
| **Parser → SplitCode** | 多对多：解析出的商品查询拆零配置决定数量分配 |

### 4.4 核心数据流

```
用户操作 → 前端上传 → 后端接收 → 模板路由 → 文件解析
    ↓
提取头部信息（order_no, receiver_org, supplier_org, receiver_name, phone, address）
    ↓
提取商品明细（item_code, item_name, quantity, unit, spec, category, remark）
    ↓
查询拆零配置（split_map）
    ↓
根据模板和拆零规则分配数量到对应列
    ↓
填充 OMS 模板 → 保存文件 → 返回下载链接 → 记录日志
```
